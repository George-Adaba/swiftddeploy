## Architecture Overview

SwiftDeploy is a **declarative infrastructure automation tool** that converts a single manifest file into a fully functional containerized deployment. Think of it as Terraform for microservices: write configuration once, and the tool handles all the complexity.

### System Design

```
┌─────────────────────────────────────────────────────────────────┐
│                    SwiftDeploy CLI Tool                         │
│                   (Python application)                          │
└─────────────────────────────────────────────────────────────────┘
                              │
                    ┌─────────┴─────────┐
                    ▼                   ▼
         ┌──────────────────┐  ┌──────────────────┐
         │  manifest.yaml   │  │  Generators      │
         │  (Single Source  │  │  - Nginx         │
         │   of Truth)      │  │  - Docker        │
         │                  │  │  - Env vars      │
         └──────────────────┘  └──────────────────┘
                                       │
                    ┌──────────────────┼──────────────────┐
                    ▼                  ▼                  ▼
            ┌──────────────┐   ┌──────────────┐   ┌──────────────┐
            │  nginx.conf  │   │ docker-      │   │   .env       │
            │              │   │ compose.yml  │   │              │
            └──────────────┘   └──────────────┘   └──────────────┘
                    │                  │                 │
                    └──────────────────┼─────────────────┘
                                       ▼
                            ┌──────────────────────┐
                            │  Docker Daemon       │
                            │  - Nginx Container   │
                            │  - App Container     │
                            │  - Shared Network    │
                            └──────────────────────┘
```

### High-Level Flow

```
User runs: ./swiftdeploy deploy
     ↓
1. Load manifest.yaml
     ↓
2. Validate manifest structure
     ↓
3. Generate nginx.conf (from template)
     ↓
4. Generate docker-compose.yml (from template)
     ↓
5. Generate .env file
     ↓
6. Run: docker-compose up --build
     ↓
7. Health checks on both services
     ↓
8. Success! Services running
```

## DevOps Decisions Explained

### 1. **Declarative Configuration (manifest.yaml)**

**Decision**: Use a single YAML manifest as the source of truth instead of manually editing multiple files.

**Why**:
- **Reproducibility**: Delete containers and re-run `swiftdeploy init` → identical setup
- **Auditability**: Track changes to one file (manifest.yaml) in git
- **Reduced Errors**: No manual typos in docker-compose or nginx configs
- **Infrastructure as Code**: Version control and review deployments like code

**Example**:
```yaml
services:
  image: swift-deploy-1-node:latest  # Single image used for both
  port: 3000                         # Port inside container
  version: "1.0.2"                   # Injected as env var
  mode: stable                       # Can be "stable" or "canary"
```

### 2. **Nginx as Reverse Proxy (Never Expose App Directly)**

**Decision**: All traffic flows through Nginx; app port never exposed directly to host.

**Why**:
- **Security**: No direct access to application; Nginx provides a security layer
- **Centralized Logging**: All HTTP traffic captured in one place
- **Header Management**: X-Mode header forwarded from app to client
- **Error Handling**: JSON error responses (502/503/504) generated by Nginx
- **Timeout Control**: Configurable via manifest for different environments

**Flow**:
```
Client → http://localhost:8080 (Nginx) → http://app:3000 (Inside Network)
                                               ↓
                                    (App never visible to client)
```

### 3. **Docker Network Isolation**

**Decision**: Use a named bridge network (`swiftdeploy-net`) instead of default bridge.

**Why**:
- **DNS Resolution**: Containers can reach each other by name (`app:3000`)
- **Network Isolation**: Only our containers on this network
- **Multi-Host Scalability**: Named networks enable Swarm mode deployment later
- **Service Discovery**: No need for hardcoded IPs

### 4. **Health Checks on Both Services**

**Decision**: Define healthcheck for app and Nginx with specific intervals.

**Why**:
- **Docker Orchestration**: Docker restarts unhealthy containers automatically
- **Deployment Blocking**: `deploy` command waits for health ✅ before continuing
- **App Health**: Checks `/healthz` endpoint (FastAPI responds with uptime)
- **Nginx Health**: Checks if Nginx is accepting requests (wget test)
- **Cascading Dependencies**: Nginx won't start until app is healthy (depends_on)

```yaml
healthcheck:
  test: ["CMD", "curl", "-f", "http://localhost:3000/healthz"]
  interval: 30s
  timeout: 5s
  retries: 3
  start_period: 5s
```

### 5. **Non-Root User in Container**

**Decision**: Run app as `appuser` (unprivileged user) instead of root.

**Why**:
- **Security**: Limits damage if container is compromised
- **Industry Standard**: All production containers run as non-root
- **Dropped Capabilities**: `cap_drop: ALL` removes unnecessary Linux kernel privileges
- Only required capability: `NET_BIND_SERVICE` (bind to network ports)

**In Dockerfile**:
```dockerfile
RUN useradd -m appuser
USER appuser
```

### 6. **Restart Policy: unless-stopped**

**Decision**: Containers restart automatically unless explicitly stopped.

**Why**:
- **High Availability**: If app crashes, Docker restarts it
- **Manual Control**: `docker-compose down` stops them permanently
- **Better than always**: Doesn't restart after user stops containers
- **Production Ready**: Mirrors real-world orchestration systems

### 7. **Named Volumes for Logs**

**Decision**: Use Docker volumes to persist Nginx and app logs.

**Why**:
- **Data Persistence**: Logs survive container restarts
- **Debugging**: Check logs from failed deployments
- **Volume Management**: `docker volume ls` lists all logs
- **Cleanup**: `docker volume prune` removes old logs safely

```yaml
volumes:
  app_logs:
    driver: local
  nginx_logs:
    driver: local
```

### 8. **Canary/Stable Deployment Mode**

**Decision**: Single image, environment variable switches mode.

**Why**:
- **Testing**: Deploy canary version before rolling out to stable
- **A/B Testing**: Run different versions simultaneously (with multiple services)
- **Minimal Overhead**: One env var, no image rebuild
- **Rolling Updates**: `promote canary` → `promote stable` with zero downtime

**Canary Features**:
- Adds `X-Mode: canary` header to every response
- Enables `/chaos` endpoint for chaos engineering (simulate failures)
- Clients can detect they're hitting a test version

### 9. **Timeout Configuration**

**Decision**: Configurable proxy timeouts in manifest (default 30s).

**Why**:
- **Slow Requests**: Some endpoints need more time (file uploads, data processing)
- **Long Polling**: Timeouts too short cause dropped connections
- **Configurable**: Change without rebuilding containers
- **All Three Phases**: connection, send, and read timeouts all set

```yaml
proxy_connect_timeout 30s;   # Time to establish connection
proxy_send_timeout 30s;      # Time to send request body
proxy_read_timeout 30s;      # Time to read response
```

### 10. **Lightweight Alpine Images**

**Decision**: Use `nginx:alpine` instead of `nginx:latest`.

**Why**:
- **Size**: Alpine is ~20MB vs ~200MB for full nginx:latest
- **Security**: Fewer packages = smaller attack surface
- **Fast Deployments**: Smaller images = faster pulls
- **Resource Efficiency**: Less disk space on your machine

## Component Breakdown

### Component 1: manifest.yaml (Configuration Source)

**File**: `manifest.yaml`

**Purpose**: Single source of truth for all deployment settings.

**Structure**:
```yaml
services:                      # Application service configuration
  image: swift-deploy-1-node:latest
  port: 3000                   # Internal port (inside container)
  version: "1.0.2"             # Injected as APP_VERSION env var
  restart_policy: unless-stopped
  health_check:
    interval: 30s              # How often to check health
    timeout: 5s                # How long to wait for response
    retries: 3                 # Fail after 3 consecutive failures
    start_period: 5s           # Grace period after container start

nginx:                         # Reverse proxy configuration
  image: nginx:alpine          # Using lightweight Alpine version
  port: 8080                   # External port (what clients connect to)
  proxy_timeout: 30            # Seconds before timeout

network:                       # Docker network configuration
  name: swiftdeploy-net        # Name for inter-container communication
  driver_type: bridge          # Bridge network (isolated from other containers)

mode: stable                   # Can be "stable" or "canary"
```

**How It's Used**:
- Swiftdeploy CLI reads this file
- Each generator (nginx, docker-compose) pulls values from it
- Promotes (`canary` ↔ `stable`) update only this file

### Component 3: Docker Compose (Container Orchestration)

**File**: `docker-compose.yml` (generated from template)

**What It Defines**:

1. **App Service**:
   - Builds from `./api/Dockerfile`
   - Sets environment: `MODE`, `APP_VERSION`, `APP_PORT`
   - Runs as non-root user
   - Has health check
   - Logs volume

2. **Nginx Service**:
   - Uses pre-built `nginx:alpine` image
   - Mounts generated `nginx.conf`
   - Has health check (wget test)
   - Depends on app being healthy
   - Exposes port 8080

3. **Network**:
   - Named `swiftdeploy-net`
   - Type: bridge
   - Both containers on same network

4. **Volumes**:
   - `app_logs`: Persists app logs
   - `nginx_logs`: Persists nginx logs

**Generated Example**:
```yaml
version: '3.8'

services:
  app:
    build: ./api
    image: swift-deploy-1-node:latest
    environment:
      - MODE=stable
      - APP_VERSION=1.0.2
      - APP_PORT=3000
    depends_on: [nginx]
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/healthz"]
      interval: 30s
      timeout: 5s
      retries: 3

  nginx:
    image: nginx:alpine
    ports:
      - "8080:8080"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
    depends_on:
      app:
        condition: service_healthy

networks:
  swiftdeploy-net:
    driver: bridge
```

### Component 4: Nginx Configuration (Reverse Proxy)

**File**: `nginx.conf` (generated from template)

**What It Does**:

1. **Listen on Port 8080**: All external traffic comes here
2. **Forward to App**: Proxies to `http://app:3000` (via DNS on swiftdeploy-net network)
3. **Logging**: Custom format captures request details
4. **Timeouts**: Configurable via manifest (prevents hanging requests)
5. **Error Pages**: Returns JSON on 502/503/504
6. **Headers**: 
   - Adds `X-Deployed-By: swiftdeploy`
   - Forwards `X-Mode` from app (canary header)
   - Sets proxy headers (Host, X-Real-IP, X-Forwarded-*)

**Generated Config Sections**:

```nginx
# Log format captures:
# - Timestamp (ISO 8601)
# - HTTP status code
# - Request duration
# - Upstream server address
# - Request URI and sizes
log_format custom '$time_iso8601 | $status | ${request_time}s | $upstream_addr | $request_uri ...';

# Timeouts prevent hanging connections
proxy_connect_timeout 30s;
proxy_send_timeout 30s;
proxy_read_timeout 30s;

# Buffering allows proper header handling
proxy_buffering on;
proxy_buffer_size 4k;
proxy_buffers 24 4k;

# Error page handlers
error_page 502 @error_502;
location @error_502 {
    default_type application/json;
    return 502 '{"error": "Bad Gateway", "code": "502", ...}';
}

# Main proxy location
location / {
    proxy_pass http://app:3000;
    add_header X-Deployed-By swiftdeploy always;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_pass_header X-Mode;  # Forward canary header
}
```

**Security Features**:
- No direct access to app
- Logs all requests centrally
- Handles errors gracefully (JSON responses)
- Enforces timeouts (prevents DOS)

#### GET /healthz
**Purpose**: Liveness probe for Docker health checks

**Response**:
```json
{
  "status": "ok",
  "uptime": 42
}
```

**Code**:
```python
start_time = time.time()  # Record when app started

@app.get('/healthz')
```

**Why Important**:
- Docker calls this every 30 seconds
- If it fails 3 times in a row, container is restarted
- Deployment waits for this endpoint before declaring success

#### POST /chaos
**Purpose**: Chaos engineering - simulate failures for testing

**Request**:
```json
{
  "mode": "slow",
  "duration": 5
}
```

**Responses**:

1. **Slow mode**: Adds N second delay before responding
   ```python
   if command.mode == "slow":
       chaos_config["mode"] = "slow"
       chaos_config["duration"] = command.duration
       # GET / will call: await asyncio.sleep(5)
   ```

2. **Error mode**: Return 500 on ~X% of requests
   ```python
   if command.mode == "error":
       chaos_config["rate"] = 0.5  # 50% error rate
       # GET / will randomly throw 500 errors
   ```

3. **Recover**: Cancel chaos
   ```python
   if command.mode == "recover":
       chaos_config["mode"] = "recover"  # Back to normal
   ```

**Example Usage**:
```bash
# Enable 50% error rate
curl -X POST http://localhost:8080/chaos \
  -H "Content-Type: application/json" \
  -d '{"mode": "error", "rate": 0.5}'

# Next requests have 50% chance of 500 error
# Useful for testing error handling

# Recover
curl -X POST http://localhost:8080/chaos \
  -H "Content-Type: application/json" \
  -d '{"mode": "recover"}'
```

```python
class ManifestLoader:
    """Parse and validate manifest.yaml"""
    
    @staticmethod
    def load() -> Dict[str, Any]:
        # Check if file exists
        if not MANIFEST_FILE.exists():
            raise FileNotFoundError(f"manifest.yaml not found at {MANIFEST_FILE}")
        
        # Open and parse YAML
        with open(MANIFEST_FILE, 'r') as f:
            manifest = yaml.safe_load(f)  # Convert YAML → Python dict
        
        return manifest
    
    @staticmethod
    def save(manifest: Dict[str, Any]):
        # Convert Python dict → YAML
        with open(MANIFEST_FILE, 'w') as f:
            yaml.dump(manifest, f, default_flow_style=False, sort_keys=False)
            # default_flow_style=False: Pretty print (multiline, not compact)
            # sort_keys=False: Keep original key order
```

```python
class Validator:
    @staticmethod
    def check_all(manifest: Dict[str, Any]) -> tuple[bool, list[str]]:
        # Define 5 checks as (name, function) pairs
        checks = [
            ("manifest.yaml exists and is valid YAML", Validator.check_manifest),
            ("All required fields present and non-empty", lambda: Validator.check_required_fields(manifest)),
            ("Docker image exists locally", lambda: Validator.check_docker_image(manifest)),
            ("Nginx port is not already bound", lambda: Validator.check_port_available(manifest['nginx']['port'])),
            ("Generated nginx.conf is syntactically valid", lambda: Validator.check_nginx_syntax()),
        ]
        
        # lambda: anonymous function
        # lambda: Validator.check_required_fields(manifest)
        # → pass manifest without calling it immediately
        
        results = []
        all_pass = True
        
        # Execute each check
        for check_name, check_func in checks:
            try:
                result = check_func()  # Call the function (True/False)
                status = "✅ PASS" if result else "❌ FAIL"
                results.append(f"{status} | {check_name}")
                if not result:
                    all_pass = False
            except Exception as e:
                results.append(f"❌ FAIL | {check_name}: {e}")
                all_pass = False
        
        # Print all results
        for result in results:
            print(result)
        
        return all_pass, results  # (bool, list)
```

```python
@staticmethod
def check_port_available(port: int) -> bool:
    try:
        # Try to connect to localhost:port
        sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        # AF_INET: IPv4
        # SOCK_STREAM: TCP (connection-oriented)
        
        result = sock.connect_ex(('127.0.0.1', port))
        # connect_ex: Connect without raising exception
        # Returns 0 if connection succeeds (port is IN USE)
        # Returns non-zero if connection fails (port is AVAILABLE)
        
        sock.close()
        return result != 0  # True if port is available
    except Exception:
        return True  # If we can't determine, assume available
```

```python
@staticmethod
def deploy(manifest: Dict[str, Any]):
    """Initialize, bring up stack, and wait for health checks"""
    print("🚀 Starting deployment...\n")
    
    # Step 1: Init (generate configs)
    DeploymentManager.init(manifest)
    
    # Step 2: Bring up containers
    print("📦 Bringing up containers...")
    try:
        result = subprocess.run(
            ['docker-compose', 'up', '--build', '-d'],
            # Command: docker-compose up --build -d
            # up: Start services
            # --build: Rebuild images from Dockerfile
            # -d: Detached (background)
            
            cwd=PROJECT_ROOT,  # Run in project directory
            capture_output=True,  # Capture stdout/stderr
            timeout=300,  # Max 5 minutes
            text=True  # Return strings (not bytes)
        )
        if result.returncode != 0:
            # Non-zero exit code = failure
            print(f"❌ Docker Compose failed: {result.stderr}")
            return False
        print("  ✓ Containers started")
    except Exception as e:
        print(f"❌ Error starting containers: {e}")
        return False
    
    # Step 3: Wait for health checks
    print("🏥 Waiting for health checks...")
    app_port = manifest['services']['port']  # 3000
    max_retries = 60  # 60 seconds timeout
    
    for i in range(max_retries):
        try:
            # Try to reach /healthz
            result = subprocess.run(
                ['curl', '-f', f'http://localhost:{app_port}/healthz'],
                # -f: Fail on HTTP error codes
                # Endpoint must be running and return 2xx status
                capture_output=True,
                timeout=5
            )
            if result.returncode == 0:
                # Success! Health check passed
                print("  ✓ Service is healthy")
                print("✅ Deployment successful\n")
                return True
        except Exception:
            pass  # Connection failed, try again
        
        if i < max_retries - 1:
            time.sleep(1)  # Wait 1 second before retry
    
    # If we get here, all 60 retries failed
    print("❌ Health check timeout after 60s")
    return False
```

```python
@staticmethod
def promote(mode: str, manifest: Dict[str, Any]):
    """Switch deployment mode (canary/stable)"""
    if mode not in ['canary', 'stable']:
        print(f"❌ Invalid mode: {mode}. Must be 'canary' or 'stable'")
        return False
    
    current_mode = manifest.get('mode', 'stable')  # Default to stable
    
    if current_mode == mode:
        print(f"ℹ️  Already in {mode} mode")
        return True
    
    print(f"🔄 Promoting to {mode} mode...\n")
    
    # Step 1: Update manifest
    manifest['mode'] = mode
    ManifestLoader.save(manifest)  # Write updated manifest
    print(f"  ✓ Updated manifest.yaml (mode: {mode})")
    
    # Step 2: Regenerate docker-compose.yml with new MODE
    docker_compose = DockerComposeGenerator.generate(manifest)
    DockerComposeGenerator.write(docker_compose)
    print("  ✓ Regenerated docker-compose.yml")
    
    # Step 3: Update .env file
    env_content = EnvGenerator.generate(manifest)
    EnvGenerator.write(env_content)
    print("  ✓ Updated .env")
    
    # Step 4: Restart only the app service (not nginx)
    print("  ⏳ Restarting service...")
    try:
        subprocess.run(
            ['docker-compose', 'restart', 'app'],
            # restart: Stop and start container
            # Only restarts 'app' service, not nginx
            cwd=PROJECT_ROOT,
            capture_output=True,
            timeout=30
        )
        print("  ✓ Service restarted")
    except Exception as e:
        print(f"  ❌ Failed to restart service: {e}")
        return False
    
    # Step 5: Verify the mode is active
    print("  ⏳ Verifying mode...")
    app_port = manifest['services']['port']
    
    for _ in range(30):
        try:
            # Call /healthz to verify service is up
            result = subprocess.run(
                ['curl', '-s', f'http://localhost:{app_port}/healthz'],
                # -s: Silent (no progress)
                capture_output=True,
                timeout=5,
                text=True
            )
            if result.returncode == 0:
                response = json.loads(result.stdout)  # Parse JSON
                if response.get('status') == 'ok':
                    print("  ✓ Mode verified")
                    print(f"✅ Promoted to {mode} mode\n")
                    return True
        except Exception:
            pass
        
        time.sleep(1)  # Try again after 1 second
    
    print("❌ Failed to verify mode")
    return False
```

```python
def main():
    """CLI command dispatcher"""
    if len(sys.argv) < 2:
        print_usage()
        sys.exit(1)
    
    # Get the subcommand (init, deploy, etc)
    command = sys.argv[1]
    
    # Load manifest for all commands except help
    if command not in ['--help', '-h']:
        try:
            manifest = ManifestLoader.load()
        except FileNotFoundError as e:
            print(f"❌ {e}")
            sys.exit(1)
    
    # Dispatch to appropriate function
    if command == 'init':
        DeploymentManager.init(manifest)
        sys.exit(0)
    
    elif command == 'validate':
        all_pass, results = Validator.check_all(manifest)
        sys.exit(0 if all_pass else 1)
    
    elif command == 'deploy':
        success = DeploymentManager.deploy(manifest)
        sys.exit(0 if success else 1)
    
    elif command == 'promote':
        if len(sys.argv) < 3:
            print("❌ Usage: ./swiftdeploy promote [canary|stable]")
            sys.exit(1)
        mode = sys.argv[2]
        success = DeploymentManager.promote(mode, manifest)
        sys.exit(0 if success else 1)
    
    elif command == 'teardown':
        clean = '--clean' in sys.argv
        success = DeploymentManager.teardown(clean)
        sys.exit(0 if success else 1)
    
    else:
        print_usage()
        sys.exit(1)

if __name__ == "__main__":
    main()
```
